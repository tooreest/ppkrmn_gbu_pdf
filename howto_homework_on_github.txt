Мой порядок действий при работе с github для выполнения и сдачи ДЗ.
На сайте github.com регистрируем учетную запись %username
Cоздаем репозиторий для выполнения и сдачи ДЗ %repname
На ПК создаем папку для репозиториев переходим в нее или работаем в домашней папке
mkdir ~/geekbrains
cd ~/geekbrains
git clone git@github.com:%username/%repname.git  # Клонируем репозиторий из github (если на ПК еще нету репозитория)
cd %repname  # Переходим в папку с репозиторием

git pull origin master  # Получаем (обновляем) версию ветки master с github (если на ПК есть репозиторий)
# если на github есть другие ветки, и они нужны, получаем аналогично, меняя их название.

git branch QxxLyy  # Создаем branch для очередного ДЗ Q - четверть, L - урок.
echo 'master step 2020xxxx' >> stepfile  # Делаем изменение в файле
git branch  # Проверяем, что мы находимся на ветке master
git commit -am 'master step 2020хххх'  # Делаем коммит в master, чтобы увести его на шаг вперед
git push origin master  # Отправляем изменения в ветке master на github
git checkout QxxLyy  # Переключаемся на созданный branch
# Выполняем ДЗ
git commit -am 'Четверть xx. Урок yy. Домашнее задание выполнено.'  # Делаем коммит после выполнения ДЗ или промежуточные.
git push origin QxxLyy  # Отправляем из резельтат на github

Для сдачи ДЗ через P[pullrequest на сайте github.com:
Переходим в репозиторий %repname
Нажимаем кнопку 'New pull resquest'
Слева оставляем master, справа изменяем на QxxLyy и нажимаем кнопку Создать 
Переходим в Pull requests
Переходим во нужный pull request
Копируем ссылку из адресной строки браузера, вставляем в строку для сдачи ДЗ на сайте geekbrains.ru

После проверки ДЗ преподователем делаем marge pull request в ветку master (Можно не делать, тогда мастер всегда будет пустой, а для каждого ДЗ отдельная ветка)
